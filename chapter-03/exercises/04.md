**Q**: The following sequence of code has been observed in various programs:
```c
  dup2(fd, 0);
  dup2(fd, 1);
  dup2(fd, 2);
  if (fd > 2)
    close(fd);
```
To see why if test if needed, assume that `fd` is `1` and draw a picture of what
happens to the three descriptor entries and the corresponding file table entry
with each call to `dup2`. Then assume that `fd` is 3 and draw the same picture.

**A**:
# `fd` is equal to 1

## dup2(fd, 0) called
- if fd 0 is open then it is closed
- then fd is duplicated and dup2 returns 0

## dup2(fd, 1) called
- fd is equal to 1, so dup2 does nothing and returns the value of fd

## dup2(fd, 2) called
- if fd 2 is open then it is closed
- then fd is duplicated and dup2 returns 2

## if (fd > 2) close(fd)
- the condition is not true so fd is not closed

# `fd` is equal to 3

## dup2(fd, 0) called
- if fd 0 is open then it is closed
- then fd is duplicated and dup2 returns 0

## dup2(fd, 1) called
- if fd 1 is open then it is closed
- then fd is duplicated and dup2 returns 1

## dup2(fd, 2) called
- if fd 2 is open then it is closed
- then fd is duplicated and dup2 returns 2

## if (fd > 2) close(fd)
- the condition is true so fd is closed
